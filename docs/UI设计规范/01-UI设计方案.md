# 摩塔 Mota - UI 设计方案

## 1. 设计概述

### 1.1 设计愿景

摩塔 Mota 的 UI 设计旨在打造一个**专业、高效、友好**的团队协作管理平台界面。通过清晰的视觉层次、一致的交互模式和现代化的设计语言，为用户提供流畅的使用体验。

### 1.2 设计原则

| 原则 | 描述 | 实践指南 |
|-----|------|---------|
| **清晰性** | 信息层次分明，用户能快速理解界面内容 | 使用明确的视觉层次、合理的留白、清晰的标签 |
| **一致性** | 相同功能保持相同的视觉和交互模式 | 统一的组件库、一致的交互反馈、规范的布局 |
| **高效性** | 减少用户操作步骤，提升工作效率 | 快捷操作、智能默认值、批量处理能力 |
| **友好性** | 界面亲和力强，降低学习成本 | 友好的提示、平滑的过渡、适当的引导 |
| **专业性** | 体现企业级产品的专业感和可信度 | 精致的细节、稳重的配色、规范的排版 |

### 1.3 设计目标

```
┌─────────────────────────────────────────────────────────────────┐
│                        设计目标矩阵                               │
├─────────────────┬─────────────────┬─────────────────────────────┤
│    用户体验      │    品牌形象      │       技术实现              │
├─────────────────┼─────────────────┼─────────────────────────────┤
│  • 操作流畅      │  • 专业可信      │  • 组件化开发               │
│  • 学习成本低    │  • 现代简洁      │  • 响应式适配               │
│  • 信息清晰      │  • 品牌一致      │  • 性能优化                 │
│  • 反馈及时      │  • 差异化识别    │  • 可维护性                 │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

## 2. 色彩系统

### 2.1 主色调

摩塔采用蓝色作为主色调，传达专业、可信、高效的品牌形象。

| 色彩名称 | 色值 | 使用场景 |
|---------|------|---------|
| **Primary** | `#2b7de9` | 主要按钮、链接、选中状态、品牌标识 |
| **Primary Hover** | `#1a6dd6` | 主色悬停状态 |
| **Primary Light** | `#e6f0fc` | 主色浅色背景、选中项背景 |
| **Primary Dark** | `#1560c0` | 主色深色变体 |

```css
:root {
    --primary-color: #2b7de9;
    --primary-hover: #1a6dd6;
    --primary-light: #e6f0fc;
    --primary-dark: #1560c0;
}
```

### 2.2 功能色

| 色彩类型 | 色值 | 使用场景 |
|---------|------|---------|
| **成功色** | `#52c41a` / `#10b981` | 成功提示、完成状态、正向趋势 |
| **警告色** | `#faad14` / `#f59e0b` | 警告提示、注意事项、中等优先级 |
| **错误色** | `#ff4d4f` / `#ef4444` | 错误提示、删除操作、高优先级 |
| **信息色** | `#1890ff` / `#3b82f6` | 信息提示、帮助说明 |

```css
:root {
    --success-color: #10b981;
    --warning-color: #f59e0b;
    --error-color: #ef4444;
    --info-color: #3b82f6;
}
```

### 2.3 中性色

| 色彩名称 | 色值 | 使用场景 |
|---------|------|---------|
| **文字主色** | `#1f2329` | 标题、正文、重要信息 |
| **文字次色** | `#646a73` | 次要文字、描述信息 |
| **占位符色** | `#8f959e` | 输入框占位符、禁用文字 |
| **禁用色** | `#bbbfc4` | 禁用状态文字 |

```css
    --bg-light: #f5f7fa;
```

### 2.5 色彩使用规范

```
┌─────────────────────────────────────────────────────────────────┐
│                        色彩使用层级                               │
├─────────────────────────────────────────────────────────────────┤
│  Level 1: 主色调 - 核心操作、品牌元素                              │
│  Level 2: 功能色 - 状态反馈、语义表达                              │
```

- 正文文字与背景对比度 ≥ 4.5:1
- 大号文字与背景对比度 ≥ 3:1
- 图形元素与背景对比度 ≥ 3:1

## 3. 字体系统

### 3.1 字体家族

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 
             'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', 
             Helvetica, Arial, sans-serif;
```

**字体优先级说明：**
1. 系统默认字体（-apple-system, BlinkMacSystemFont）
2. Windows 系统字体（Segoe UI）
3. 中文字体（PingFang SC, Hiragino Sans GB, Microsoft YaHei）
4. 通用无衬线字体（Helvetica Neue, Helvetica, Arial）

### 3.2 字号规范

| 级别 | 字号 | 行高 | 使用场景 |
|-----|------|-----|---------|

| H2 | 24px | 1.4 | 页面副标题、区块标题 |
| H3 | 18px | 1.5 | 卡片标题、模块标题 |
| H4 | 16px | 1.5 | 小标题、列表标题 |
| Body | 14px | 1.6 | 正文内容 |
| Small | 13px | 1.5 | 辅助文字、描述信息 |
| Caption | 12px | 1.5 | 标签、时间戳、提示 |
| Mini | 11px | 1.4 | 徽章、极小文字 |

### 3.3 字重规范

| 字重 | 数值 | 使用场景 |
|-----|------|---------|
| Regular | 400 | 正文、描述文字 |
| Medium | 500 | 按钮文字、导航项 |

**行高规则：**
- 标题类：1.2 - 1.4
- 正文类：1.5 - 1.6
- 多行文本：1.6 - 1.8

- 段落间距：16px - 24px

**字间距：**
- 正文：默认

## 7. 动效系统

### 7.1 过渡时间

| 级别 | 时长 | 使用场景 |
|-----|------|---------|
| fast | 150ms | 按钮状态、图标变化 |
| normal | 300ms | 面板展开、页面切换 |
| slow | 500ms | 复杂动画、引导动画 |

### 7.2 缓动函数

| 类型 | 函数 | 使用场景 |
|-----|------|---------|
| ease | `ease` | 通用过渡 |
| ease-in | `ease-in` | 元素退出 |
| ease-out | `ease-out` | 元素进入 |
| ease-in-out | `ease-in-out` | 状态切换 |

```css
:root {
    --transition-fast: 0.15s ease;
    --transition-normal: 0.3s ease;
    --transition-slow: 0.5s ease;
}
```

### 7.3 常用动画

**淡入动画：**
```css
@keyframes fadeIn {
    from { opacity: 0; }
    to { opacity: 1; }
}
```

**向上滑入：**
```css
@keyframes slideUp {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}
```

**缩放进入：**
```css
@keyframes scaleIn {
    from {
        opacity: 0;
        transform: scale(0.9);
    }
    to {
        opacity: 1;
        transform: scale(1);
    }
}
```

## 11. 响应式设计

### 11.1 断点定义

| 断点名称 | 宽度范围 | 设备类型 |
|---------|---------|---------|
| xs | < 576px | 手机竖屏 |
| sm | 576px - 767px | 手机横屏 |
| md | 768px - 991px | 平板 |
| lg | 992px - 1199px | 小型桌面 |
| xl | 1200px - 1399px | 标准桌面 |
| 2xl | ≥ 1400px | 大型桌面 |

### 11.2 响应式策略

**移动端适配（< 992px）：**
- 侧边栏隐藏，通过汉堡菜单触发
- 顶部搜索栏隐藏
- 统计卡片改为 2 列或 1 列
- 表格支持横向滚动

**平板适配（768px - 991px）：**
- 侧边栏可折叠
- 内容区域自适应
- 部分组件简化显示

**桌面端适配（≥ 992px）：**
- 完整布局显示
